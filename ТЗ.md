# Технічне завдання: Smart Trading Assistant / Personal AI Broker Assistant

> Це початкове ТЗ проєкту. README.md описує, що реалізовано,
> AGENTS.md — як продовжувати. Цей файл зберігає повні вимоги-орієнтир.

## 1. Роль
Спроєктувати й реалізувати готовий продукт: простий зовні, але дуже розумний
всередині автоматизований ШІ-помічник для торгівлі та інвестицій.

Це НЕ «казино-бот». Це система, яка: автоматично аналізує ринки; використовує
велику базу знань з книг, правил трейдерів, брокерів, інвесторів і risk
management; враховує новини; приймає рішення на основі багатьох факторів;
контролює ризик; працює з малим капіталом; показує live-графіки; накопичує
досвід; навчається на статистиці; має простий україномовний інтерфейс.

**Принцип: зовні — максимально просто, всередині — глибокий аналіз, велика
база знань, статистика, навчання і контроль ризику.**

## 2. Головна мета
Автоматизований помічник для поступового росту капіталу. Користувач вносить
невелику суму (100/300/500/1000 USD), натискає «Почати стратегію», бачить
наживо роботу системи, баланс, прибуток/збиток/ризик/угоди/графіки, натискає
«Зупинити і проаналізувати», отримує звіт українською.

Мета — не «100→1000 за 2 місяці» ціною ризику всього депозиту, а: зберегти
капітал; не допустити великих втрат; заробляти поступово; накопичувати
статистику; покращувати правила; будувати реальну перевагу.

## 3. Філософія
Пріоритети: виживання капіталу; контроль ризику; якісні рішення; дисципліна;
статистичне навчання; поступове реінвестування; довгострокове покращення.

Система НЕ повинна: martingale; подвоювати після збитку; «відіграватися»;
торгувати заради активності; ризикувати великою частиною рахунку; надмірне
плече; входити без stop-loss; ігнорувати погані дані; обіцяти прибуток.

Система повинна: чекати якісні можливості; не торгувати без переваги;
пояснювати кожну дію; зупинятись при підвищеному ризику; накопичувати знання;
перевіряти висновки статистикою; відрізняти перевагу від випадковості.

## 4. Сценарій
Простий україномовний dashboard: баланс, P/L, активні/закриті угоди, денний/
тижневий ризик, просадка, статус, що робить система і чому, що вивчила,
live-графіки. Кнопки: Почати стратегію / Зупинити і проаналізувати / Пауза /
Закрити всі угоди.

## 5. Режими
- Analysis Mode — лише аналіз, без угод.
- Paper Trading Mode — симуляція на реальних даних (комісії, slippage, затримки).
- Live Trading Mode — реальна торгівля малим капіталом. За замовчуванням
  вимкнено; вмикається лише після перевірки й явного підтвердження.

## 6-7. Автоматизація + україномовний інтерфейс
Мінімум ручної роботи, дисциплінована консервативна автоматизація під
контролем risk engine. Увесь інтерфейс українською, природною простою мовою,
складні терміни коротко пояснені. Без оманливих кнопок («гарантований прибуток»).

## 8-9. Trading Knowledge Constitution + Rule Evolution
Велика внутрішня база узагальнених правил (без копіювання текстів книг) з
повною структурою: id, назва, категорія, джерело, опис, коли застосовується/ні,
підтвердження, ризики, які помилки запобігає, приклад, failure conditions,
confidence, ефективність, статус, дата перегляду. Статуси: Core Safety, Strong,
Useful, Context-Dependent, Uncertain, Weak, Deprecated, Dangerous. Core Safety
не вимикаються автоматично. Правила еволюціонують за власною статистикою,
обережно, без радикальних висновків з малої вибірки.

## 10. Market Data + Data Quality
Ціни, обсяги, історія, order book, волатильність, ліквідність, spreads тощо.
Замінні джерела: Binance, Kraken, Bybit, Yahoo, Alpha Vantage/Polygon, CSV/JSON.
Data Quality Engine перевіряє пропуски, аномалії, затримки, конфлікти джерел.
Якщо дані ненадійні — не торгувати.

## 11. News Engine
Збирати й аналізувати новини: важливість, настрій (позитив/негатив/нейтрал/шум),
сила впливу, короткостроковий хайп vs фундаментальна зміна, чи ринок уже
врахував. Ranking довіри джерел: офіційні/регулятори/ЦБ/великі медіа/біржі/
соцмережі/чутки. Низькоякісні джерела не є достатньою підставою для угоди.

## 12-13. Technical + Fundamental Analysis
Технічний: тренд, S/R, MA, EMA/SMA, RSI, MACD, ATR, Bollinger, обсяг, breakout,
market structure. Не єдина підстава для угоди. Фундаментальний: для акцій
(виручка, прибуток, маржа, борг, P/E, P/S, growth); для крипти (ліквідність,
обсяг, tokenomics, funding, регуляторні ризики).

## 14. Signal Engine
Не «купити/продати», а структурована ідея: asset, direction, horizon, entry
logic, why now, фактори за/проти, stop-loss, take-profit, invalidation, size,
risk/reward, confidence, сценарії, причина відмови. Угода дозволена лише за
чіткою логікою, зі стопом, R:R не нижче мінімуму, з кількома підтвердженнями,
якісними даними, без суперечності новин, у відповідному режимі ринку.

## 15. Risk Engine (найважливіший)
Має право заблокувати будь-яку угоду. За замовчуванням Conservative Growth:
ризик 0.25–0.5% на угоду, денний 1%, тижневий 2–3%, просадка до паузи 5–8%,
без плеча, spot first, лише ліквідні активи. Блокує при відсутності стопу,
поганому R:R, перевищенні лімітів, неліквідності, поганих даних, API-помилках,
loss streak, надмірній кореляції. Emergency Stop при лімітах втрат/просадки,
некоректному API, ненадійних даних, аномальній поведінці.

## 16-17. Portfolio Manager + Reinvestment
Аналіз усього портфеля (ризик, кореляція, концентрація, ліквідність, risk
budget). Реінвестування 0/25/50/75/100% (за замовчуванням не 100%), з показом
впливу на ріст/волатильність/просадку. Не збільшувати ризик через випадковий
успіх.

## 18-20. Learning + Investment Memory + 80/20
Після кожної угоди зберігати повний контекст. Learning Engine визначає що дає
прибуток/збиток, реальну перевагу vs шум, без радикальних висновків з малої
вибірки. Investment Memory зберігає ринкові ситуації, патерни, реакції, з
confidence/статусом; слабкі спостереження не стають жорсткими правилами.
80/20 Engine знаходить 20% факторів, що дали 80% результату.

## 21-23. Cycles + Backtesting + Paper
Короткий цикл 2-3 міс (перші дані), довгий 12 міс (різні режими ринку).
Backtesting перед live: total return, win rate, drawdown, Sharpe, Sortino,
profit factor, expectancy, з уникненням overfitting. Paper на реальних даних.

## 24. Live Trading Integrations
Binance основна, потім Kraken, Bybit. Adapter відокремлений від логіки аналізу:
баланс, ордери, стопи, статус, error handling, retry, rate limits, key safety,
dry-run, sync, emergency close.

## 25. Payments/Withdrawal Planner
Документувати маршрути виводу (Binance Wallet, Trustee, PayPal, Skrill,
Payoneer): найпростіший шлях, комісії, ризики, час, KYC/AML, чи підходить.

## 26-28. Dashboard + Reports + Journal
Простий dashboard (баланс, equity curve, P/L, позиції, drawdown, win rate,
метрики, ризик, ranking, режим ринку, статус, дія, пояснення). Звіти
українською (щоденний, тижневий, після Stop з 80/20 і планом). Journal кожної
ідеї й угоди з повним контекстом.

## 29-34. Reality Check, Strategy Lab, Scanner, Research, Explainability, Confidence
Оцінка реалістичності цілі. Лабораторія стратегій (backtest+paper перед live).
Opportunity Scanner з ранжуванням. Research summary перед важливою угодою.
Пояснення кожної дії українською. Confidence обережно, не як гарантія,
калібрований на історії.

## 35-37. Security + Architecture + Tech Stack
Безпечна робота з ключами (.env, не логувати, read-only для аналізу, мінімальні
права, withdrawal вимкнено, audit logs, emergency stop). Модульна замінна
архітектура, локальний запуск. Стек: Python FastAPI / Node TS; React/Next;
PostgreSQL; time-series; job scheduler; Redis/черги; chart library; exchange
adapter; тести; Docker.

## 38-41. MVP + Acceptance + Self-Review + Головна вимога
MVP: україномовний dashboard, Start/Stop, watchlist, market data, індикатори,
news, Constitution, Signal Engine, Risk Engine, Paper Trading, Live skeleton,
Binance, Journal, Reports, Learning v1, Investment Memory v1, 80/20 v1,
onboarding, config, тести, документація.

Acceptance: локальний запуск, українська, Start/Stop, dashboard, live charts,
paper, risk engine, journal, reports, knowledge, learning, memory, Binance
adapter, безпека ключів, документація, тести, self-review; система не обіцяє
прибутку, блокує небезпечні угоди, зрозуміла без фінансової освіти.

Self-Review: створити Master Requirements, провести аудит (Complete/Partial/
Missing), виправляти, мінімум 5 циклів.

**Головна вимога: не складність заради складності, не Bloomberg, не азартний
бот. Будувати простого україномовного автоматизованого ШІ-помічника для
поступового росту капіталу, що глибоко аналізує ринок, використовує базу
знань, контролює ризик, навчається, показує все наживо. Якщо вибір між швидким
прибутком і збереженням капіталу — обирати збереження капіталу.**

**Головний продукт: Start. Watch live. Stop. Review. Improve. Repeat.**
